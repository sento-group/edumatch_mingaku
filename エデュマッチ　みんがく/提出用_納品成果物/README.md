# エデュマッチ（みんがく）リニューアル - システム設計段階 納品成果物

## 概要

本フォルダには、エデュマッチ（みんがく）リニューアルプロジェクトのシステム設計段階における納品成果物をまとめています。

## 重要事項

**これらの成果物は「たたき台」として提供されます**

- 完璧な最終版ではなく、開発進行に伴い更新・最適化される設計資料です
- クライアントの要件ヒアリングや実際の開発過程で、より具体的で実用的な内容に発展させていきます
- 初期段階での認識齟齬を防ぎ、共同で最適解を見つけていくための基盤資料としてご活用ください

## 納品成果物一覧

### 1. プロジェクト概要書
プロジェクト全体の方向性と範囲を明確化した資料です。
- プロジェクト目標・背景
- 想定する解決課題
- プロジェクト制約事項
- スケジュール概要
- HTMLモックに反映されている機能の整理
- 技術・インフラの詳細
- データ移行・運用計画

### 2. ステークホルダーマップ
関係者の役割と責任範囲を可視化した資料です。
- 利用者層の分類（管理者、一般ユーザー等）
- 各ステークホルダーの権限レベル
- 承認フローの明確化
- コミュニケーション窓口の整理

### 3. 業務フロー図（As-Is / To-Be）
現在の業務とシステム導入後の変化を明示した資料です。
- 現状業務プロセス（As-Is）
- システム導入後の業務プロセス（To-Be）
- 改善ポイントの明示
- 業務時間短縮効果の試算

### 4. 画面一覧・画面遷移図
ユーザーの操作イメージを具体化した資料です。
- 全画面リスト（機能別分類）
- 画面遷移の流れ
- 各画面の主要機能
- 権限による表示制御
- HTMLモックアップ（52画面）

### 5. 機能一覧（概要）
システムの主要機能を整理した資料です。
- 機能一覧（優先度付き）
- 各機能の概要説明
- 実装優先順位
- 段階的リリース計画

### 6. 開発スケジュール案
開発の進捗計画を可視化した資料です。
- フェーズ別開発期間（目安：18-24週間）
- 中間確認ポイント（要件確定、アルファ版、ベータ版）
- リリース段階計画
- ユーザー受け入れテスト計画
- 週単位の詳細スケジュール（目安）

### 7. 予算見積もり
投資額の目安を明示した資料です。
- 開発費用内訳（概算1,155万円・税込）
- インフラ費用（月額・概算）
- 保守費用（年額・概算）
- 段階的投資プラン（検討事項）
- 初年度総額（概算1,289-1,337万円）

### 8. 移行スケジュール概要書
既存サイトからの引継ぎ要素と移行手順の俯瞰資料です。
- 引継ぎ対象一覧（ドメイン/DNS/SSL/サーバ/DB/画像/ドキュメント/設定）
- データ移送方法と依存関係
- 環境差分と整備項目
- カットオーバー手順/ロールバック計画
- リスクと対策
- 高レベル移行スケジュール

## 閲覧方法

1. **まず**: `00_納品成果物一覧.md` を開いて全体構成を確認してください
2. **次に**: `01_プロジェクト概要書.md` を開いてプロジェクト全体を把握してください
3. **その後**: 各成果物を順番に閲覧してください

## HTMLモックアップの閲覧方法

`04_画面設計/HTMLモックアップ/index.html` をブラウザで開くと、全52画面の一覧が表示されます。
各画面をクリックすると、詳細なモックアップを確認できます。

## 注意事項

- 本成果物は概算・想定に基づいており、詳細設計後に精緻化されます
- 数値・日付・期間は「概算」「想定」「目安」であり、協議の上で確定します
- HTMLモックアップに反映されている機能は、実装可否を要検証です
- 詳細仕様は協議の上で確定します
