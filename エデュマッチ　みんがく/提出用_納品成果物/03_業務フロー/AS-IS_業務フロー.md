# エデュマッチ（既存ユーザー）AS-IS 業務フロー

> **本ドキュメントは「たたき台」として提供されます**  
> 完璧な最終版ではなく、開発進行に伴い更新・最適化される設計資料です。

## 1. 目的

本ドキュメントは、既存ユーザーがエデュマッチを利用して問い合わせに至るまでの現状フローを可視化したものです。TO-BE業務フローとの比較により、改善ポイントを明確化することを目的としています。

## 2. 業務フロー概要

以下は、既存ユーザーがエデュマッチを利用して問い合わせに至るまでの現状フローです。

```mermaid
flowchart TD
  A[課題の顕在化\n- 新規塾/新コース立ち上げ\n- 既存のリニューアル] --> B[解決策の情報収集\nWeb検索/比較検討]
  B --> C{教育業界の比較サイトは少ない}
  C -->|レビューが充実| D[エデュマッチを発見/閲覧]
  D --> E[レビュー/ランキング/事例を閲覧]
  E --> F{良さそうか?}
  F -->|YES| G[気になるサービスの\nツール詳細ページへ]
  F -->|NO| B
  G --> H["問い合わせ"ボタンを押下]
  H --> I{会員登録は完了しているか?}
  I -->|NO| J[会員登録フローへ遷移]
  I -->|YES| L[問い合わせフォーム入力]
  J --> K[メール認証/基本情報登録\n(登録完了)]
  K --> L[問い合わせフォーム入力]
  L --> M[問い合わせ送信\n(事業者へ連携)]
  M --> N[事業者からの返信待ち/比較継続]
```

## 3. フローの特徴

### 3.1 主な特徴
- レビュー・ランキング・事例を閲覧してから問い合わせに至る
- 会員登録は問い合わせ時に必要に応じて実施
- 比較的シンプルなフロー

### 3.2 課題点（検討事項）
- 会員登録への導線が弱い可能性（要検証）
- 記事閲覧と会員登録の連携が弱い可能性（要検証）
