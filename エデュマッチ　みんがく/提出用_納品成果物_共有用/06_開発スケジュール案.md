# 開発スケジュール案（たたき台・協議の上確定）

> **本ドキュメントは「たたき台」として提供されます**  
> 完璧な最終版ではなく、開発進行に伴い更新・最適化される設計資料です。  
> クライアントの要件ヒアリングや実際の開発過程で、より具体的で実用的な内容に発展させていきます。  
> 初期段階での認識齟齬を防ぎ、共同で最適解を見つけていくための基盤資料としてご活用ください。

## 前提条件

- HTMLモックアップ（52画面）は設計段階で作成済み
- 本スケジュールは概算であり、要件確定後に詳細化を予定
- 各フェーズの期間は協議の上で調整可能
- 実際の期間は要件確定後に協議の上で確定します

## フェーズ別スケジュール（目安・協議の上確定）

### フェーズ1: 要件定義・詳細設計（目安：4-6週間）

**主要作業項目（検討中）**:
- 要件ヒアリング・合意形成
- データモデル設計
- API仕様設計
- 詳細画面設計（HTMLモックをベースに）
- 非機能要件の詳細化

**成果物（想定）**:
- 要件定義書
- データモデル/ER図
- API仕様書
- 詳細画面設計書

### フェーズ2: 開発環境構築（目安：1-2週間）

**主要作業項目（検討中）**:
- インフラ構築（AWS環境）
- WordPress環境構築
- 開発環境整備
- CI/CDパイプライン構築

**成果物（想定）**:
- 開発環境構築完了
- 環境構築手順書

### フェーズ3: 実装開発（目安：8-12週間）

**主要作業項目（検討中）**:
- フロントエンド実装（HTMLモックをベースに）
- バックエンド実装
- 外部API連携
- 認証・認可機能
- データベース実装

**成果物（想定）**:
- ソースコード一式
- 単体テスト仕様書/結果報告書

### フェーズ4: テスト・品質保証（目安：3-4週間）

**主要作業項目（検討中）**:
- 単体テスト
- 結合テスト
- システムテスト
- ユーザー受け入れテスト（UAT）
- パフォーマンステスト

**成果物（想定）**:
- テスト仕様書/結果報告書
- バグ管理表
- 品質報告書

### フェーズ5: 移行・リリース（目安：2-3週間）

**主要作業項目（検討中）**:
- データ移行
- 本番環境構築
- リリース作業
- リリース後監視

**成果物（想定）**:
- 移行完了報告書
- リリースノート
- 運用手順書

## マイルストーン（目安・協議の上確定）

| マイルストーン | 時期（目安） | 内容 | 備考 |
|-------------|------------|------|------|
| 要件確定 | フェーズ1完了時 | 要件定義書の合意 | 協議の上で確定 |
| アルファ版 | フェーズ3中間時点 | 主要機能の動作確認版 | 内部検証用 |
| ベータ版 | フェーズ4開始時 | ユーザー受け入れテスト版 | 外部検証用 |
| 本番リリース | フェーズ5完了時 | 正式リリース | 協議の上で確定 |

※ 各マイルストーンの具体的な日付は、要件確定後に協議の上で確定します。

## スケジュール概要（週単位・目安）

以下は概算の週単位スケジュールです。実際の期間は要件確定後に協議の上で確定します。

- **週1-2**: IA/デザイン / 旧→新URLマップ / テーマ基盤構築を想定
- **週3-4**: テンプレート実装（トップ/記事/サービス/検索/詳細）を想定
- **週5**: データ移行試行 / 301マップ検証 / SEO設定を想定
- **週6**: 総合テスト / UAT / コンテンツ差し替えを想定
- **週7**: リリース / 監視 / チューニングを想定

## 注意事項

- 各期間は目安であり、実際の期間は要件確定後に協議の上で確定します
- リスクや課題が発生した場合、スケジュールの見直しが必要になる可能性があります
- 詳細スケジュールは要件確定後に精緻化します
- 本スケジュールは[プロジェクト概要書.md](./01_プロジェクト概要書.md)の§9と連動しています

## 依存関係

- 要件確定が完了してから詳細設計を開始
- 詳細設計が完了してから実装開発を開始
- 実装開発が完了してからテストを開始
- テストが完了してから移行・リリースを開始
