## 1. 目的

本ドキュメントは、エデュマッチ（みんがく）リニューアル後の業務フローを可視化したものです。AS-IS業務フローとの比較により、改善ポイントを明確化することを目的としています。

## 2. 業務フロー概要

想定する新体験：記事や比較ページからの「全文はこちら」CTAで会員登録に自然導線を設けることを検討。登録後はプラン比較を経て、問い合わせ/資料請求/キープが可能になることを想定しています。

```mermaid
flowchart TD
  A[課題の顕在化\n- 新規塾/新コース立ち上げ\n- リニューアル] --> B[情報収集\n検索/広告/紹介]
  B --> C[エデュマッチの記事/紹介記事/レビューへ流入]
  C --> D[記事の要約/冒頭を閲覧]
  D --> E[全文はこちらボタン\n(会員登録前ゲート)]
  E --> F{会員登録する?}
  F -->|YES| G[会員登録フロー\n(メール/SSO/基本情報)]
  F -->|NO| C
  G --> H[プラン選択画面\n(無料/有料プラン比較)]
  H --> I{有料を選択?}
  I -->|YES| J[決済フロー\n(カード/請求書)]
  I -->|NO| K[無料プランで開始]
  J --> L[有料プランアクティベート]
  K --> L[利用開始]
  L --> M[レビュー全文/比較機能/ランキング詳細\nキープ・資料請求が解放]
  M --> N{気になるサービスは見つかった?}
  N -->|YES| O[サービス詳細\n(仕様/価格/事例)]
  N -->|NO| P[レコメンド/カテゴリ再探索]
  O --> Q[資料請求/問い合わせボタン]
  Q --> R[問い合わせフォーム入力\n(要件テンプレ/自動補完)]
  R --> S[送信\n(事業者・CSに連携)]
  S --> T[ダッシュボードで進捗確認/比較継続]
```

## 3. フローの特徴（検討中）

### 3.1 主な特徴（検討中）
- 記事の冒頭のみ公開し、全文閲覧には会員登録を促す導線を検討
- プラン選択（無料/有料）を経てから利用開始を想定
- マイダッシュボードでの進捗確認・比較継続機能を検討

### 3.2 改善ポイント（検討中）
- 会員登録への自然な導線の追加
- プラン選択による機能の段階的解放
- ダッシュボードによる利用状況の可視化

### 3.3 注意事項
- プラン選択はアンカー表現であり、正式仕様は協議の上で確定します
- 各機能の実装可否は要検証です
- 詳細仕様は要件確定後に精緻化します

## 4. AS-ISとの比較

### 4.1 主な変更点（検討中）
- **会員登録のタイミング**: 問い合わせ時 → 記事閲覧段階へ前倒しを検討
- **プラン選択の追加**: 無料/有料プランの選択フローを追加（検討中）
- **ダッシュボード機能**: 進捗確認・比較継続機能を追加（検討中）

### 4.2 期待される効果（試算・要検証）
- 会員登録率の向上（要検証）
- 有料プランへの転換率向上（要検証）
- ユーザーエンゲージメントの向上（要検証）
